# GenOS Knowledge Capture

GenOS 현장 작업(FDE)에서 *한 번 배운 걸 다음 세션·다음 고객사에서 다시 파헤치지 않도록* 만드는 개인 스킬이다. AI의 세션 기억은 끊기지만, vault에 원자적 노트로 남기고 **MOC(허브)에서 엮어두면** 새 세션이 그 한 장만 읽고 빠르게 온보딩한다.

이 스킬은 사용자 개인 자산(`~/.claude/skills/`)이다. **회사 레포의 `.claude/skills/genos-*`(genos-session-close 등)는 회사 공유 자산이라 절대 건드리지 않는다.** 세션 상태는 회사 HANDOFF가, 재사용 지식은 이 스킬이 담당한다 — 둘은 분리된 관심사다.

## 핵심 원칙 (왜 이렇게 하나)

온보딩 가속의 실체는 "그래프뷰가 예쁜 것"이 아니라 **작은 진입점(MOC)을 먼저 읽고, 거기 링크 중 오늘 필요한 1~2개만 선택적으로 읽는 것**이다. AI는 Obsidian 그래프를 순회하지 못한다. 그래서 이 스킬은 늘 두 가지를 함께 한다: ① 원자적 노트 작성, ② **MOC에 링크 추가**. MOC에 안 걸린 노트는 새 세션이 발견하지 못하는 고아가 된다.

토큰 절감은 "재탐색(re-grep·재독)을 대체할 때 + 선택적으로 1~2개만 읽을 때"만 생긴다. 그러니 노트는 **재사용 가능한 일반화(패턴·함정·결정)**만 담아야 한다. 오늘 무슨 파일을 고쳤는지 같은 휘발성 상태를 vault에 넣으면 노이즈만 쌓이고 검색 가치가 떨어진다.

## 3계층 — 무엇을 캡처하고 무엇을 버리나

캡처 후보가 생기면 먼저 어느 계층인지 판별한다. 계층을 섞지 않는 것이 vault가 비대해져도 길을 잃지 않는 유일한 규칙이다.

| 계층 | 무엇 | 저장 위치 | 캡처? |
|------|------|----------|-------|
| **플랫폼** | GenOS 자체 구조·패턴·함정 (모든 고객사 재사용) | `Projects/GenonAI/GenOS/` | ✅ |
| **고객사** | 특정 고객 커스텀 결정·맥락 (삼성카드 등) | `Projects/GenonAI/<고객사>/` 또는 `GenonAI/` loose | ✅ |
| **세션 상태** | 오늘 진행/대기/다음 작업 | vault 밖 — GenOS 레포 `HANDOFF_NOW.md` | ❌ |

판단 기준: **"이걸 6개월 뒤 다른 세션·다른 고객사에서 다시 보면 시간을 아껴줄까?"** Yes면 플랫폼/고객사 노트, No(이번 작업에만 유효)면 캡처하지 않는다.

Vault 루트: `~/Library/Mobile Documents/iCloud~md~obsidian/Documents/Note`
GenOS 영역: `<vault>/Projects/GenonAI/`

## 워크플로우

순서엔 실제 의존성이 있다(중복 확인 → 작성 → MOC 갱신). 하지만 **비협상은 둘뿐이다: ② 작성 전 중복 확인, ⑤ MOC 갱신.** 나머지는 상황에 맞게 조정해도 된다 — 후보가 하나면 계획 제시를 짧게, 명백하면 확인을 생략하는 식으로.

### 1. 후보 추출 — 이번 세션에서 일반화 가능한 것만
대화·작업 내역에서 재사용 지식을 골라낸다. 좋은 후보: 비자명한 패턴, 빠지기 쉬운 함정, 협의로 확정된 설계 결정, "문서엔 X라는데 실제론 Y"인 사실. 나쁜 후보: 진행상황, 파일 목록, 커밋 해시, 이번 한정 상태.

### 2. 계층 판별 + 중복 확인 (중요)
각 후보가 플랫폼/고객사 중 어디인지 정한다. 그다음 **기존 노트를 먼저 검색**한다:
```bash
ls "<vault>/Projects/GenonAI/GenOS/" "<vault>/Projects/GenonAI/"
grep -rl "<핵심 키워드>" "<vault>/Projects/GenonAI/"
```
같은 주제 노트가 있으면 **새로 만들지 말고 그 노트를 갱신**한다(원자성 유지, 중복 금지).

### 3. 사용자에게 캡처 계획 제시 → 확인
글을 쓰기 전, 무엇을 어디에 쓸지 짧게 보여주고 확인받는다. 사용자는 위임 엔지니어 방식을 선호하므로 길게 묻지 말고 목록만:
```
[캡처 계획]
- GenOS/<제목>.md (플랫폼·신규)  — <한 줄 요지>
- <고객사>/<제목>.md (고객사·기존 갱신) — <무엇 추가>
- MOC: "패턴·함정" 섹션에 링크 추가
```
확인 후 작성. 사용자가 "그냥 해" 류면 확인 생략 가능.

### 4. 노트 작성 — Zettelkasten 형식 (vault 기존 컨벤션 준수)
`obsidian-note` 스킬과 동일 형식. **단, Tavily 웹 리서치는 생략한다** — 출처는 웹 개념이 아니라 이 세션의 실작업·레포다.

Frontmatter:
```yaml
---
source:
  - "local:/Users/sskim/work/GenOS/..."   # 또는 "youtrack:SAMSUNGCARD-96"
related_notes:
  - "[[실존하는_노트만]]"                  # 반드시 vault에 존재하는 노트
tags:
  - work/genos
  - type/pattern        # 또는 type/moc, type/onboarding
  - customer/samsung-card  # 고객사 노트일 때
  - feature/<기능>         # 횡단 검색용
created: YYYY-MM-DD        # 오늘 날짜
---
```
본문:
- **## 핵심 아이디어** — 정의가 아니라 hook/통찰 한 문단. "왜 이게 중요한가".
- 주제에 맞는 자유 섹션 — **trade-off와 판단을 반드시 포함**. 근거(코드 `file:line`, 실측 수치)로 뒷받침. 위키피디아 톤 금지, 엔지니어링 블로그 톤.
- **## 더 알아보기** — 구체적 후속 질문 (키워드 나열 금지).

파일명: 순수 제목 `.md`. GenOS 플랫폼 노트는 vault 관행대로 `GenOS ` 접두어 포함(예: `GenOS admin-api 외부 시스템 로그 연동 패턴.md`). 고객사 노트는 고객사명으로 시작(예: `삼성카드 PIMS 행위로그 연동.md`).

원자성: **한 노트 = 한 아이디어.** 쪼갤 수 있으면 쪼갠다. 대화의 번호 항목을 옮길 땐 일부를 압축·생략하지 말고 동일 깊이로.

### 5. MOC 갱신 (생략 금지)
`<vault>/Projects/GenonAI/GenOS/GenOS 지식 MOC.md`를 열어 적절한 섹션(코드 온보딩 / 운영·장애 / 패턴·함정 / 고객사별 / 학습)에 `- [[새 노트]] — 한 줄 요지` 추가. 새 노트의 frontmatter `related_notes`에도 `[[GenOS 지식 MOC]]`를 넣어 양방향으로 엮는다.

### 6. 결과 보고
작성/갱신한 노트 목록과 MOC 변경을 요약한다.

## Gotchas — 여기서 실제로 틀린다

이 스킬을 실행할 때 빠지기 쉬운 함정들. 규칙이 아니라 *왜 틀리는지*로 기억하라.

- **MOC 갱신을 빼먹는다 (1번 실패 모드).** 노트만 쓰고 만족하기 쉽다. 하지만 MOC에 링크를 안 걸면 새 세션은 그 노트의 존재 자체를 모른다 — AI는 그래프뷰를 못 보고 MOC만 진입점으로 읽기 때문이다. *작성한 노트는 반드시 MOC에 한 줄 추가하고 끝낸다.*
- **중복 노트를 새로 만든다.** 비슷한 주제가 이미 있는데 검색을 건너뛰고 새 파일을 만들면, 지식이 두 곳에 갈라져 둘 다 신뢰를 잃는다. 작성 전 `grep -rl`로 먼저 찾고, 있으면 갱신한다.
- **세션 상태를 지식으로 착각한다.** "오늘 chat_service 3줄 고쳤다"는 패턴이 아니라 상태다. "6개월 뒤 다른 고객사에서 시간을 아껴줄까?"를 통과 못 하면 캡처하지 말고 HANDOFF로 보낸다.
- **존재하지 않는 노트를 `[[링크]]`한다.** related_notes의 모든 wikilink는 vault에 실존하는 노트 + 이번에 함께 만드는 노트여야 한다. 미존재 링크는 깨진 그래프를 만든다.
- **고객사 노트 위치가 혼재한다.** 현재 삼성카드 노트는 `GenonAI/` loose에 있다. 한 고객사 노트가 3장 이상 쌓이면 `GenonAI/<고객사>/` 폴더 분리를 *제안*하되, 기존 파일 이동은 사용자 확인 후에만 — 임의로 옮기면 다른 노트의 `[[링크]]`가 깨질 수 있다.
- **회사 자산을 건드린다.** 캡처 자동화가 아쉬워도 회사 레포 `.claude/skills/genos-*`는 절대 수정하지 않는다. 모든 자동화는 이 개인 스킬(`~/.claude/`) 안에서 끝낸다.

## 예시

**예시 1 — 플랫폼 패턴 캡처**
세션에서 "외부 시스템 전송은 BackgroundTasks + dry-run 플래그로 한다"는 패턴을 적용함.
→ `GenOS/GenOS admin-api 외부 시스템 로그 연동 패턴.md` (tags: `type/pattern`, `backend/integration`), MOC "패턴·함정"에 링크.

**예시 2 — 고객사 결정 캡처**
"삼성카드 고객관리번호는 CST_MNGT_NO로 넘어오고, 컬럼은 삼성카드 DB에만 추가"가 확정됨.
→ `삼성카드 PIMS 행위로그 연동.md` 기존 노트 **갱신**(중복 생성 X), tags `customer/samsung-card` `feature/pims`.

**예시 3 — 캡처 거부**
"오늘 chat_service_service.py 3줄 고치고 커밋함" → 세션 상태. **캡처하지 않고** HANDOFF로 안내.
